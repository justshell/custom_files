Описание:
--------------

Независимое управление файлами, директориями, симлинками на хосте Linux

Пример описания для хоста
--------------
#### Общий вид
```
  custom_files:                           # Начало определения управляющего dict
    internal:                             # dict для адаптации внутренней логики
      template_path: <path>               # где производить поиск шаблонов (Default: 'custom_files')
    dirs:                                 # Секция объявления директория
      <keyname>:                          # Произвольное имя ключа
        path: '<path>'                    # Путь на хосте, где создается директория (без вложений)
        owner: '<linux_user>'             # Владелец директории (Default: 'root')
        group: '<linux_group>'            # Группа для директории (Default: <owner> or 'root')
        mode: '<ansible_filemode>'        # Права для директории (Default: '0755')
    files:                                # Секция объявления файлов
      <keyname>:                          # Произвольное имя ключа
        state: '<touch|absent|file>'      # Touch - создать пустой файл, absent - удалить, file - сгенерировать файл (Default: 'file')
        src: '<path>'                     # Используется если state==file, указывается:
                                          #   - источник шаблона для генерирования файла ( путь от директории custom_files )
                                          #   - url источника, в этом случае источник не воспринимается, как шаблон (сохраняется "как есть").
                                          #                    Источник должен отвечать вменяемо на запросы HEAD/GET.
                                          #                    check-mode при этом показывает, что будет изменение (особенность использованного модуля get_url)
                                          #                    но при этом их (изменений) может не быть при реальной прокатке
                                          # При использовании URL допускается указать checksum и checksum_algorithm (чтобы не производить сетевые запросы без необходимости)
                                          # Поддерживаемые алгоритмы: 'md5', 'sha1', 'sha256', 'sha512'
                                          # При этом check-mode отображает информацию в привычном человеческом виде, а также происходит валидация скачанного файла
        dest: '<path>'                    # Путь на хосте целевого файла
        owner: '<linux_user>'             # Используется если state=='file' or 'touch', установка владельца для файла (Default: 'root')
        group: '<linux_group>'            # Используется если state=='file' or 'touch', установка группы для файла (Default: <owner> or 'root')
        mode: '<ansible_filemode>'        # Используется если state=='file' or 'touch', установка режима доступа к файлу (Default: 0644)
    symlinks:                             # Объявление симлинков
      <keyname>:                          # Произвольное имя ключа
        src: <path>                       # Путь на хосте, куда ссылается симлинк
        dest: <path>                      # Путь на хосте, где создается симлинк
```

Пример playbook
--------------
```
---
- hosts: linux
  become: True
  become_method: sudo
  roles:
    - custom_files
```
Запуск playbook
--------------
```
ansible-playbook -i inventory -l <some_host> playbooks/custom_files.yml
```
